# 客户讲稿

这份讲稿按页面切换顺序组织，控制在 3 分钟左右。讲解时只保留关键结论，不展开底层实现细节。

## 1. 项目说明页

“这个系统的目标不是简单展示一条车流动画，而是把车辆怎么运行、区间状态怎么变化、异常怎么传播这三件事放在同一个平台里看清楚。我们用仿真把道路运行过程还原出来，再把结果沉淀成可回放、可分析、可预测的数据。”

“从业务角度看，它主要解决三个问题：第一，快速复现典型交通场景；第二，解释异常和拥堵是怎么形成的；第三，为后续预警和策略验证提供依据。”

## 2. 场景模板页

“这里是场景配置入口。我们可以直接组合流量、车道数、道路长度、天气、异常比例和门架位置，把不同业务场景参数化，而不是每次重新搭建。”

“这一步的意义是让系统从‘固定演示’变成‘可复用模板’。同一套平台可以切换不同运行条件，方便做对比分析。”

## 3. 仿真控制页

“这里是系统的实时运行中心。左侧配置参数，中间看路网和车辆状态，右侧和下方看统计、图表和日志。”

“仿真启动后，系统按时间步推进，车辆会根据跟驰和换道规则自动运行，门架、异常、排队和预警也会同步生成。也就是说，页面上看到的每一个结果，都来自同一条实时仿真链路。”

## 4. 可视回放页

“这一页展示的是历史结果回放。和普通报表不同，这里可以按时间重看车辆怎么走、异常在哪一刻出现、拥堵是怎么扩散的。”

“对客户来说，这一页最重要的价值是可追溯。不是只告诉你‘发生了什么’，而是能把过程完整还原出来。”

## 5. 预警仪表盘和工作流页

“这一部分是告警和策略能力。仪表盘负责汇总异常、趋势和关键指标，工作流页负责把规则配置出来，让不同告警条件和动作可视化、可维护。”

“这意味着系统不只是能看，还能判断和响应。我们可以把经验规则固化为工作流，让预警逻辑更容易调整和验证。”

## 6. 预测工作台和态势大屏

“最后两页对应的是数据沉淀和展示闭环。预测工作台会把历史运行结果整理成训练数据，并支持模型训练和评估；态势大屏则把关键结果统一展示，方便运营视角查看整体运行状态。”

“所以这套系统完整的链路是：场景配置、实时仿真、历史回放、规则预警、数据训练、态势展示。它不是单点工具，而是一套从仿真到分析再到决策支持的闭环平台。”
