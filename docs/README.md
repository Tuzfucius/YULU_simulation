# 文档索引

本文档目录用于说明 ETC 仿真系统的设计、运行机制、数据结构、接口关系与存储方案。

## 推荐阅读顺序

1. `system_working_principles.md`
   内容：系统总体架构、实时仿真、历史存储、回放、训练、分析、核心数据结构与 API。
2. `storage/README.md`
   内容：存储专题索引、历史记录分层与路径轨迹嵌入方案入口。
3. `storage/api_interaction_and_history_storage.md`
   内容：历史存储、接口迁移、兼容性与性能优化专题说明。
4. `simulation_mechanics.md`
   内容：仿真机制与模型层面的补充说明。
5. `developer_guide.md`
   内容：开发者使用与扩展指引。

## 文档分工

- `system_working_principles.md`
  作为当前系统工作原理总览文档，面向产品、开发与重构设计。
- `storage/`
  聚焦运行历史、回放、训练数据提取、路径轨迹嵌入、存储性能优化。
- 其他文档
  聚焦特定子系统或开发流程，不重复描述全局架构。

## 当前重点

当前系统已经从“单个大结果文件驱动”逐步演进为“`run_id` + 摘要 + 分析 + 回放 + 分层存储”模式。阅读文档时，建议优先理解以下三条主线：

1. 实时仿真如何生成运行结果。
2. 历史结果如何以 `run_id` 为中心组织和读取。
3. 分析、回放、训练如何复用同一份历史数据。
