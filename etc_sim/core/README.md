# Core - 核心业务逻辑

该目录包含了不依赖于 Web 框架的纯仿真业务逻辑组件。

## 文件说明

- **vehicle.py**: 
  - `Vehicle` 类定义：封装了单辆车的物理属性和决策逻辑。
  - **核心模型**: 实现了 IDM (跟驰模型) 和 MOBIL (换道模型)。
  - **状态管理**: 负责处理车辆的异常产生逻辑（如故障导致的非正常减速）。
  - **ETC 检测**: 处理车辆通过 ETC 门架时的检测时间记录。

## 逻辑细节

- **IDM 计算**: 根据前车状态和自身目标速度，计算下一时刻的期望加速度。
- **MOBIL 决策**: 综合考虑变道后的利益（加速度增益）和其对后车的影响，决定是否执行变道。
- **碰撞/受影响判定**: 根据与异常车辆的距离和速度差，判定当前车辆是否受到交通流波动的影响。
